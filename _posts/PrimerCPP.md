# Chapter 12 Class     （Anstract and Encapsulation） #
## 12.1   类的定义和声明 ##
  C++中，类可以用struct和class来声明，类的生命和定义一般都在头文件中。再累内部定义的函数默认为inline  

  在类外部定义的成员函数必须指定是在类的作用域中。Person::Person(){}  

  成员函数后面加const可已经成员函数声明为常量，不能改变其操作的数据成员，const关键字必须同时出现在声明和定义中
  
  **向前声明**（forward declaration,不完全类型）可以用于定义该类的指针和引用  

  C++中将struct视为类，所以可以省略struct关键字
          
## 12.2   this指针 ##
  const对象只能使用const成员; 非const对象可以使用任何成员，但非const版本是更好的匹配. 
 
  const函数只能返回*this作为const引用  

  任意成员函数（包括const成员）都可以改变mutable对象的值  
## 12.3 类作用域 ##
  函数局部作用域(形参)->类作用域（Screen::height）->外围作用域（::height）
## 12.4 构造函数 ##
  构造函数初始化式 : 初始化阶段+普通的计算阶段  
  可以初始化const对象或者应用类型的对象，但不能对他们赋值  

    ConstRef::ConstRef(int ii):i(ii),ci(ii),ri(ii){}  
 
  初始化列表仅仅指定值，不指定顺序，初始化的顺序就是定义的顺序  
  构造函数中常常制定默认实参 

    Sales_item(const std::string &book=" "):isbn(book),revenue(0.0){} 
 
  只有当一个类没有定义构造函数时，编译器才会自动生成一个默认构造函数。默认构造函数只对定义在全局作用域中的对象才初始化，对于定义在局部作用域中的内置或复合类型的成员不初始化。  

  **隐式转换**：用单个实参来定义的构造函数定义了从形参类型到该类类型的一个隐式转换。  

    item.same_isbn("99-99");//使用接受一个字符串的Sales_item构造函数生成一个新的（临时的）Sales_item对象 
 
   1. 构造函数声明为explicit，抑制隐式转换  
   2. 为了转换而显式地使用构造函数
  对于没有定义构造函数且全体成员均为public的类，可以用字面量初始化（一般只于struct）

## 12.6 static类成员：独立于该类的任意对象而存在，每个static数据成员是与类相关联的对象，并不与该类的对象相关联 ##
  static 数据成员必须在类定义体的外部定义
  
  随类的定义而只加载一次，和Java一样，不能在非static函数中定义和调用。一般在静态环境中定义（代码块和函数体外）。
  
  整型的static const成员可以在类定义时直接初始化，在类的外部定义时不必指定初始值（实际检验也可以不再定义）。
  
  和其他成员一样，static成员未定义直接使用会引发链接错误。

# Chapter 15 面向对象编程 #
## 15.1 概述 ##
  1. **继承**（inheritance）：派生类（derived class）能够继承基类（base class）定义的成员，可以不改变与派生类具体特性不相关的操作，可以重新定义那些与派生类型相关的成员函数，将函数特化。还可以定义更多的成员。
  2. **动态绑定**（dynamic binding）：使用继承层次中任意类型 的对象，无需关心对象的具体类型，无需区分函数是在基类还是在派生类中。
  3. 通过基类的**引用（或指针）**调用虚函数时将发生动态绑定。用引用或指针调用的虚函数在运行时确定，被调用的函数是引用或指针所指对象的实际类型所定义的。
## 15.2 定义基类和派生类 ##
  virtual启用动态绑定。对于非虚函数的调用在编译时确定。

  除了构造函数之外，任何非static函数都可以是虚函数。

  基类通常应该将派生类需要重新定义的任意函数定义为虚函数

  protected成员可以被派生类对象访问但不能被普通用户访问。派生类只能通过派生类对象访问其基类的protected成员，对基类对象的protected成员没有特殊访问权限。基类对象是基类的普通用户，不能访问基类的protected成员

  类派生列表: class classname: access-label base-class

  派生类一般会重定义所继承的虚函数，否则使用基类中定义的版本

  每个派生类对象都有其基类部分，类可以访问其基类 的public和protected成员，就好像是自己的成员一样
###15.2.4 virtual与其他成员函数 ###
  要触发动态绑定，必须满足两个条件 

1. 只有指定为虚函数的成员函数才能进行动态绑定，非虚函数不进行动态绑定 
2. 必须通过基类类型对象的引用或指针进行函数调用

  关键点在于静态类型（在编译时可以知道的引用类型或指针类型）和动态类型（指针或引用所绑定的对象的类型，运行时可知）

  如果调用非虚函数，则永远执行基类类型所定义的函数。如果调用虚函数，则直到运行时才确定调用哪个函数

  只有通过引用或者指针调用，虚函数才在运行时确定。（复制构造函数不支持动态绑定）

  非虚函数总是在编译时根据对象、引用或指针确定，即使在派生类中定义了同名函数，也会调用基类中的版本（由静态类型确定）

  覆盖虚函数机制并强制函数调用虚函数的特定版本，可以使用作用域操作符。一般只有在成员函数的代码中才使用作用域操作符覆盖虚函数机制

###15.2.5 公有、私有和受保护的继承
  派生类可以进一步限制但不能放松对所继承的成员的访问

  对类所继承的成员的访问由基类中的成员访问级别和派生类列表中使用的访问标号共同控制

  公用继承(public inheritance):基类成员各自保持自己的访问级别

  受保护继承(protected inheritance):基类的public和protected成员在派生类中为protected成员

  私有继承(private inheritance):基类中所有成员在派生类中为private成员

  1. 接口继承与实现继承：public派生类继承基类的接口 private和protected派生的类不继承基类的接口，这些派生通常被称为实现继承
  2. 去除个别成员 using Base::size; 派生类可以恢复继承成员的访问级别，但不能改变基类中指定的访问级别
  3. 默认继承保护级别：使用class定义的派生类默认具有private继承，用struct定义的类默认具有public继承
###15.2.6 友元关系不继承：
1. 基类的友元对派生类没有特殊访问权限 
2. 如果基类被授予友元关系，基类的派生类也不能访问授予友元关系的类

## 15.5 继承情况下的类作用域 ##
###      15.5.1 名字查找在编译时发生（静态查找） ###
   基类类型的指针、引用或对象只能访问对象的基类部分，与实际的类型无关。

###     15.5.2 名字冲突与继承 ###
   与基类成员同名的派生类成员将屏蔽对基类成员的直接访问

   可以使用与操作符访问被屏蔽的基类成员

###  15.5.3 作用域与成员函数 ###
   1. 在基类和派生类中使用同一名字（参数列表、返回类型可以不同）的成员函数，在派生类作用域中的派生类成员将屏蔽基类成员。屏蔽的前提是基类中的函数是非虚函数，否则是错误的（虚函数的重定义必须拥有完全相同的函数原型，因为名字查找是在静态编译时做的）。
   2. 重载函数:如果派生类想通过自身类型使用所有的重载版本，则派生类或者冲定义所有的重载版本，或者一个也不重定义（否则发生函数名屏蔽）  
   为了避免这种麻烦，可以使用using声明。为基类成员函数名而作的using声明将该函数的所有重载实例加到派生类的作用域。

###  15.5.4 虚函数与作用域 ###
   确定函数调用遵循以下4个步骤： 

   1. 首先确定进行函数调用的对象、引用、指针的静态类型
   2. 在该类(静态类型)中查找函数，如果找不到，就在其基类中查找，如此循着继承链往上找
   3. 一旦找到了改名字，就进行常规类型检查，查看该函数调用是否合法
   4. 假定函数调用合法，编译器就生成代码。如果函数是虚函数或通过引用或指针调用，编译器生成代码以确定根据对象的动态类型运行哪个函数版本。否则，编译器生成代码直接调用函数

# Chapter16 模板与泛型编程 #
## 16.1 模板定义 ##
  模板定义以template开始，后接模板形参表

  类型形参跟在关键字class或者typename之后定义

  1. 使用**函数模板**（function template）时，编译器会推断哪个模板实参（template argument）绑定到模板形参(template parameter)。一旦编译器确定了实际的模板实参，就称它实例化(instantiate)了函数模板的一个实例

  2. **类模板**（class template）

  3. **模板形参**：每一个模板形参的前面都必须带上关键字class或typename

  4. **非类型模板形参**：调用函数时非类型形参将用值代替，值的类型在模板形参表中指定。在需要常量表达式的时候，可使用非类型形参
## 16.2 实例化 ##
  编译器用模板产生指定的类或者函数的特定类型版本。产生模板的特定类型实例的过程称为实例化。

  使用类模板时，必须显示指定模板实参。 Queue不是类型，Queue<string>, Queue<int>才是类型

  使用函数模板时，通常会为推断模板实参
### 16.2.1 模板实参推断 
  从函数实参确定模板实参的类型和值的过程叫做模板实参推断（template argument deduction）
         
  一般而言，不会转换实参以匹配已有的实例化，相反会产生新的实例。除了产生新的实例外，编译器只会执行两种转换

  1. const转换
  2. 数组或函数到指针的转换
          
  类型转换的限制只适用于类型为模板形参的那些实参
  
  可以使用函数模板对函数指针进行初始化或赋值

      template <typename T> int compare(const T&, const T&);
      int (&pf1) (const int&, const int&) = compare; //OK

  获取函数模板实例化的地址的时候，上下文必须是：它允许为每个模板形参确定唯一的类型或值

### 16.2.2 函数模板的显式实参
  在返回类型中使用类型形参。指定返回类型的一种方式是引入第三个模板形参，他必须由调用者显式指定

  显式模板实参从左至右与对应模板形参相匹配。加入可以从函数形参推断，则只有结尾的形参的显式模板实参可以省略。
##  16.3 模板编译模型 ##
   要进行实例化，编译器必须能够访问定义模板的源代码

   1. 包含编译模型（inclusion compilation model）
   2. 分别编译模型（separate compilation model）:export关键字 能够知名给定的定义可能会需要在其他文件中产生实例化
