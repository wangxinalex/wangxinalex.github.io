##3.6链接属性（linkage）
- external 
- internal 
- none
     
 不是在代码块中声明的变量属性默认为external， 函数属性一般也为external

 static将属性改为internal

 局部变量和形参默认为none

 extern 声明在别处定义的变量，一般用于代码块内与临时声明的局部变量相区分，一旦使用extern，就意味着该变量必须在别处被定义

 用extern的地方一般不初始化，在代码块外会警告，在代码块内同时使用extern和初始化会报错
{%highlight c%}
    int f(){
        int e;
        extern int f;//f必须在别处被定义，否则链接器报错。
    }
{%endhighlight%}

##3.8存储类型（storage class）
* static：在任何代码块之外的声明，不能指定其他的存储类型。 在代码块内部的声明如果加上static，可以使存储类型从自动变为静态，但并不改变scope。函数代码总是处在静态内存中。（static关键字代码块内部的声明改变存储类型，代码块之外改变链接属性）。只初始化一次。
* automatic：代码块内部声明，处于stack中，不会自动初始化
* register：寄存器中，不可取址，不会初始化。
