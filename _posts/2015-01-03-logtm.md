#Abstract 
1. extends a MOESI directory protocol to enable both fast conflict detection on evicted blocks and fast commit
2. handles aborts in software with little performance penalty.

#1.Introduction
![](http://i.imgur.com/kg2poCC.png)

1. TM systems must provide *atomicity* and *isolation*, requiring *version management* and *conflict detection*
	1. **Version Management:** *eager* or *lazy*
	2. **Conflict Detection:** signals an overlap between one transaction's *write set* and other concurrent transaction's *read set* or *write set*  
2. Ideally, TM should use eager version management and eager conflict detection
	1.  commits faster than aborts
	2.  eager conflict detection finds conflicts early
3.  *Log-based Transactional Memory (LogTM)*
	1.  eager version management -> per-thread *transaction log* in cacheable virtual memory
	2.  detects in-cache conflicts using a directory protocol and read/write bits on cache blocks
	3.  **On commit**: discards the log and flash clearing the read/write bits
	4.  **On abort**: walk the log to restore values
4.  Contributes
	1.  eager version management -> storing new values *in place*
	2.  allowing eviction of transactional data by extending MOESI directory protocol to enable
		1.  fast conflict detection on evicted blocks
		2.  fast commit by lazily resetting the directory state
		3.  handle aborts via a log walk with little performance penalty

#2. LogTM: Log-based Transactional Memory
##2.1 Version Management

![](http://i.imgur.com/U2vmZHs.png)

* LogTM stores old values in *per-thread log in cacheable virtual memory*
* To suppress redundant log writes, LogTM augments the state of each cached block with a *write bit* that tracks whether a block has been logged
* Since log is not needed until abort time, writes can be delayed in a *hardware log write buffer* and either discarded on a commit, or performed on an abort or when resources fill
* The principle merit of LogTM is fast commits. To commit, a processor flash clears's W bits and resets the thread's log pointer (to discard the transaction's log)
* a drawback is that aborts are slower. To abort, LogTM must *undo* the transaction by writting the old values back to their appropriate virtual addresses from the log before resetting the W bits
	* since a block may be logged more than once, *undo* must proceed from the end of log back to the beginning

##2.2 Conflict Detection
Eager conflict detection:

1. **requesting processor** sends a coherence request to the directory
2. the directory *responds* and possibly *forwards* the requests to one or more processors
3. each **responding processor** examines some local state to *detect* a conflict
4. the **responding processors** each *ack* (no conflict) or *nack* (conflict) the request
5. the **requesting processor** *resolves* any conflict
