#Abstract 
1. extends a MOESI directory protocol to enable both fast conflict detection on evicted blocks and fast commit
2. handles aborts in software with little performance penalty.

#Introduction
![](http://i.imgur.com/kg2poCC.png)

1. TM systems must provide *atomicity* and *isolation*, requiring *version management* and *conflict detection*
	1. **Version Management:** *eager* or *lazy*
	2. **Conflict Detection:** signals an overlap between one transaction's *write set* and other concurrent transaction's *read set* or *write set*  
2. Ideally, TM should use eager version management and eager conflict detection
	1.  commits faster than aborts
	2.  eager conflict detection finds conflicts early
3.  *Log-based Transactional Memory (LogTM)*
	1.  eager version management -> per-thread *transaction log* in cacheable virtual memory
	2.  detects in-cache conflicts using a directory protocol and read/write bits on cache blocks
	3.  On commit: discards the log and flash clearing the read/write bits
	4.  On abort: walk the log to restore values
4.  Contributes
	1.  eager version management -> storing new values *in place*
	2.  allowing eviction of transactional data by extending MOESI directory protocol to enable
		1.  fast conflict detection on evicted blocks
		2.  fast commit by lazily resetting the directory state
		3.  handle aborts via a log walk with little performance penalty
